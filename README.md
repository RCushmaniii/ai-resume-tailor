# AI Resume Tailor

**Live Demo:** [https://ai-resume-tailor-client.vercel.app](https://ai-resume-tailor-client.vercel.app)  
**Repository:** [github.com/RCushmaniii/ai-resume-tailor](https://github.com/RCushmaniii/ai-resume-tailor)

**Release:** v1.0.0  
**Documentation:** See sections below for complete documentation index

A full-stack web application that analyzes resumes against job descriptions using OpenAI's GPT-4, providing quantified match scores, missing keyword detection, and actionable improvement suggestions. Built to demonstrate production-ready deployment, API integration, and modern frontend architecture.

---

## The Problem

Over 75% of resumes are rejected by Applicant Tracking Systems (ATS) before reaching human recruiters. Job seekers lack visibility into why their applications fail and what specific improvements would increase their chances.

## The Solution

AI Resume Tailor provides instant, AI-powered analysis that identifies:

- **Quantified compatibility** via a 0-100 match score with granular breakdown
- **Missing keywords** ranked by priority (HIGH/MEDIUM/LOW) for ATS optimization
- **Actionable suggestions** generated by GPT-4 based on semantic understanding

**Key Design Principles:**

- Privacy-first (no data storage, real-time processing only)
- Zero-friction UX (no signup, no payment, instant access)
- Production-ready infrastructure with auto-deployment

## Technical Highlights

### 1. Environment-Based Configuration

**Challenge:** Vite bakes environment variables at build time, not runtime.  
**Solution:** Implemented proper `VITE_` prefixing and cache-busting deployment strategy.

```typescript
// Build-time injection
const API_URL = import.meta.env.VITE_API_URL || '/api';
```

**Learning:** Understanding the difference between build-time and runtime configuration is critical for modern frontend tooling.

### 2. CORS Configuration for Multi-Environment Deployments

**Challenge:** Vercel generates unique preview URLs (`*.vercel.app`) for each deployment, all blocked by CORS.  
**Solution:** Implemented wildcard pattern matching in Flask CORS config.

```python
CORS(app, resources={
    r"/api/*": {
        "origins": [
            "https://ai-resume-tailor-client.vercel.app",  # Production
            "https://*.vercel.app"  # All preview deployments
        ],
        "methods": ["GET", "POST", "OPTIONS"],
        "allow_headers": ["Content-Type"],
    }
})
```

**Learning:** Production CORS config must account for CI/CD preview environments, not just the main domain.

### 3. Cold Start UX Optimization

**Challenge:** Render's free tier spins down after inactivity, causing 30-50 second delays on first request.  
**Solution:** Implemented loading states, user education ("Waking up server..."), and optimistic UI patterns.

**Tradeoff:** Accepted cold starts for free hosting versus paying for always-on instances. Documented this clearly for stakeholders.

### 4. Input Validation & Security

**Challenge:** Prevent XSS attacks via malicious resume text.  
**Solution:** Multi-layer validation:

- Client-side: Regex patterns to detect HTML/JavaScript injection
- Server-side: Content sanitization before OpenAI API calls
- Length limits (200-10,000 characters) to prevent abuse

```typescript
const MALICIOUS_PATTERNS = [
  /<script[\s\S]*?>[\s\S]*?<\/script>/gi,
  /javascript:/gi,
  /on\w+\s*=/gi, // onclick, onerror, etc.
];
```

## Deployment

### Production URLs

- **Frontend:** https://ai-resume-tailor-client.vercel.app
- **Backend API:** https://ai-resume-tailor-hxpr.onrender.com

### Environment Variables

**Vercel (Frontend):**

```bash
VITE_API_URL=https://ai-resume-tailor-hxpr.onrender.com/api
```

**Render (Backend):**

```bash
OPENAI_API_KEY=sk-proj-xxxxx
FRONTEND_URL=https://ai-resume-tailor-client.vercel.app
```

### Deployment Pipeline

1. Push to `main` branch triggers auto-deploy on both platforms
2. Vercel builds React app with environment variables injected at build time
3. Render builds Python environment and starts Gunicorn server
4. Health check endpoints verify successful deployment

- Node.js 18+ with pnpm

```bash

git clone https://github.com/RCushmaniii/ai-resume-tailor.git
```

## Key Learnings

1. **Build vs. Runtime Config:** Vite's build-time variable injection requires careful cache management during deployment. Learned to force cache clears when environment variables change.

2. **CORS Complexity:** Production apps need CORS configs that handle preview URLs, not just the main domain. Wildcard patterns are essential for modern CI/CD.

3. **Free Tier Tradeoffs:** Documented cold start delays transparently. In a real product, I'd migrate to paid hosting; for a portfolio, the tradeoff is acceptable and demonstrates cost-conscious decision-making.

4. **API Key Security:** Never commit `.env` files. Use platform-specific environment variable managers (Vercel/Render dashboards) for production secrets.

5. **User-Friendly Errors:** Generic "Failed to fetch" messages confuse users. Implemented context-aware error handling (timeout vs. CORS vs. server errors).
